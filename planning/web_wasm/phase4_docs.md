# 計画詳細

## T4.1: ブラウザ版導線

README の最上部（タイトル直下）に以下のブロックを追加：

- 公開 URL へのリンク（バッジ風 or ボタン風）
- 1 行コピー: 「インストール不要、ブラウザで動く」
- 動作中のデモ GIF または動画（既存の README に貼られている動画を流用 or 新規撮影）

## T4.2: WASM ローカルビルド手順

README の「必要環境」「起動」セクションを以下のように再構成：

```
## 必要環境
- ネイティブ版: Rust toolchain（edition 2024）
- Web 版（ローカル）: Rust toolchain + wasm32-unknown-unknown ターゲット + trunk

## 起動
### ネイティブ
cargo run

### Web（ローカル）
rustup target add wasm32-unknown-unknown
cargo install --locked trunk
trunk serve  # http://127.0.0.1:8080
```

コマンドは Phase 1 で確定したものをそのまま使う。

## T4.3: 操作差分の注記

ネイティブ版と Web 版で異なる挙動を README 末尾の「Web 版の制約」セクションに列挙：

- フォントが OS デフォルトに置き換わる場合がある
- 初回ロードで数 MB のダウンロードが発生する
- モバイルでは修飾キーの代わりにトグルボタンを使う
- その他、Phase 2 の検証で見つかった差分

差分が無視できる程度なら本セクションは省略してよい（最終判断は Phase 4 着手時）。

## T4.4: 全体計画への反映

`planning/overall_plan.md` の「技術スタック方針」内の `[Answer]` ブロックに、WASM 配布が実装完了したことを追記する（過去経緯ではなく現状の事実として）。
