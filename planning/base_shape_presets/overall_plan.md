# Base Shape 基本図形プリセット

## 目的

手描き線に加え、代表図形をワンタップで `FractalState::base_shape` に取り込めるようにする。試行錯誤の初動を短くし、再帰変換（複製）との組み合わせをすぐに探索できるようにする。

## 計画概要

| Phase | 内容 |
|-------|------|
| [Phase 1: UI と挙動の共通枠](phase1_ui.md) | Base Shape ヘッダに「Shape」導線を追加し、一覧から選択すると線分が **追加** される。Undo・ワイド／ナロー両レイアウト対応。 |
| [Phase 2: プリセット定義と選定](phase2_presets.md) | 正規化キャンバス上の線分集合として各プリセットを定義。フラクタル探索向けのラインナップとパラメータ方針を確定する。 |

**実装スタックの前提**: レンダリング入力は Bevy、UI は `bevy_egui`。**サブメニュー**は Bevy UI ではなく egui のドロップダウン／ポップアップ／メニュー系ウィジェットで実装するのが既存コードと一貫する。

## フラクタルとして面白いプリセットの考え方

このアプリでは基図形は **折れ線の集合**であり、フラクタルは **複製変換（相似変換のリスト）の再帰適用**で生成される。プリセット選定では次を重視する。

1. **自己相似の素地**: 正多角形・正星・立方体の稜線展開など、**対称・反復しやすい外形**は、少ないレプリカで「族」が豊かに見えることが多い。
2. **有名フラクタルの「種」**: 線分 1 本（コッヒ曲線）、正三角形の辺（シェルピンスキー系）、正方形枠（カーペット／ビック系の発想）、L 字型折れ線（ドラゴン曲線やヒルベルトのモチーフの簡略版）など、ユーザーが文献イメージと結び付けやすい。
3. **再帰で境界が増える形**: **枝・亀裂・稲妻状**の折れ線は、深さを増やしたときエッジが複雑になる傾向がある。
4. **対称性のブレーク用**: わずかに **非対称**な折れ線や凹多角形は、単純な正配置レプリカでも絵が単調になりにくい。

逆に、あまり刺さらない場合: 全周が密な**塗りつぶし風の閉曲線だけ**で、開いた線分が少ないと「折れ線フラクタル」としての変化が見えにくい（当面は開多角形・星・樹枝を中心にする）。

詳細な候補一覧と優先度は [Phase 2](phase2_presets.md) に委ねる。

## 計画詳細（横断）

- **データモデル**: 現状の `BaseShape::lines` のみで表現。新しい Resource は必須ではない（プリセット ID → 線分生成関数／テーブル）。
- **操作**: 選択時は既存線に **追記**。既存の「Clear」「`-`」で整える（ワイド・ナロー両方の Base Shape ヘッダに **Clear** を用意する）。必要なら後続で「置き換えモード」を検討（本計画の受け入れ範囲外）。
- **`DrawState` とプリセット**: プリセット適用は末尾へ線を追加するだけなので、適用後も **既存の選択インデックスは維持**する。範囲外になった選択は `edit.rs` の既存処理で `Idle` に落ちる。
- **描画・編集**: 追加された線は既存の選択・移動・端点編集の対象になる。
- **正規化座標**: 既存の `[-1, 1]` 系に合わせ、プリセットは **一様スケールでキャンバス内に収まる**よう中心・外接サイズを固定する（定数は Phase 2 で決める）。

## 受け入れ条件

### Phase 1

- [ ] Base Shape ブロックのヘッダで、「`-`」ボタンの**左隣**に「Shape」導線がある（ワイド・ナロー両方）。
- [ ] 「Shape」を操作するとサブメニュー（または同等の一覧 UI）が開き、少なくとも 1 件の図形を選べる。
- [ ] 選択後、`base_shape.lines` に対応する線分が追加され、Result に反映される。
- [ ] 変更前に既存どおり `UndoStack` に積まれる。
- [ ] コンパイル・`clippy`・既存の画面操作に致命的なリグレッションがない。

### Phase 2

- [ ] ドキュメント [Phase 2](phase2_presets.md) に沿う複数プリセットが実装されている。
- [ ] 各プリセットの大きさ・中心が一貫し、極端に画面外にはみ出さない（必要なら定数調整で満たす）。
