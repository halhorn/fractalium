# Phase 1: Shape 導線とサブメニュー

## 目的

`show_canvas_block` で生成される Base Shape ヘッダに「Shape」を配置し、一覧からプリセットを選ぶと基図形に線分を追加する。ワイド・ナローともヘッダのボタン列は **[Shape] [-] [Clear]** の順（Shape を `-` の左）。ナローにもワイドと同様に **Clear** を置く。

## 設計方針

- **UI ウィジェット**: egui 標準の **`MenuButton` + メニュー項目**、**`ComboBox`**、または **`popup_below_widget` + ボタン** のいずれかで「押下 → 一覧 → 選択」を実現。Bevy ネイティブ UI は使わない（本プロジェクトのパターンに合わせる）。
- **タッチ**: メニュー項目の十分なタップ領域（既存の `step_glyph_button` 等とバランスを取る）。
- **状態**: メニュー開閉のみの持ち方は UI ローカルでよい。**プリセット適用後の `DrawState`**: プリセットは線を **末尾に追記**するため、既存の `Selected(i)` はインデックスまわりでそのまま有効。**選択は維持する**（直前に編集していた線を選んだまま）。削除などでインデックスが範囲外になった場合は、既存の `edit.rs::handle_drag_input` の out-of-bounds チェックで `Idle` に戻す。
- **重複配置**: ワイド・ナロー両方で `Base Shape` ヘッダクロージャを更新するため、ボタン列は **共通の小関数に抽出**して重複を避ける。
