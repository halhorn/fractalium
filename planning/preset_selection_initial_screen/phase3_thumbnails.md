# Phase 3: サンプル画像（サムネイル）

## 目的

各プリセットカードに、ユーザーがパターンを識別できるサンプル画像を表示する。Phase 2 のプレースホルダーを置き換え、解像度・縦横比・ダーク UI とのコントラストを揃える。

方針は `overall_plan.md` のとおり **事前 PNG 同梱**。

## 計画概要

| ID | 内容 |
|----|------|
| T1 | アセットの配置場所・命名規則・解像度・縦横比を決める。 |
| T2 | ビルドに含める手順（`trunk`/WASM を含む）を確認する。 |
| T3 | egui で画像を読み込み、`TextureHandle` 等でカードに表示する（既存の画像・フォント読み込みパターンがあれば踏襲）。 |
| T4 | 「新規」セルは空キャンバスのイメージか、専用アイコンで他プリセットと並べて一貫した枠サイズにする。 |

## 計画詳細

- **同梱 PNG**: パス規約、アスペクト比統一、`trunk`/WASM での読み込み可否を確認する。
- **アクセシビリティ**: 画像がなくてもラベルで操作可能な状態は Phase 2 済みであり、画像は追加の手がかりとして扱う。

## ファイル・ディレクトリと責務

**アセット（リポジトリ）**

| 配置（案） | 責務 |
|------------|------|
| `assets/fractalium/preset_thumbnails/`（仮。実装時に命名確定） | `FractalPreset` ごとの **事前書き出し PNG**。ファイル名規則を `fractal_presets` 側または本ディレクトリの README 相当のコメントで固定（実装者向け） |
| 新規「新規作成」用 PNG または専用アイコン | `tiles` の先頭セル専用。無ければ Phase 2 のプレースホルダー継続でも可 |

**コード（`ui/preset_picker/` への追加・変更）**

| 追加・変更 | 責務 |
|------------|------|
| `ui/preset_picker/thumbnails.rs`（新規） | **パス解決**（列挙 → アセットパス）、初回ロード、`egui::ColorImage` / `TextureHandle` キャッシュ用 `Resource`（例: `PresetThumbnailImages`）。**ビジネスロジックは持たない** |
| `ui/preset_picker/tiles.rs`（変更） | サムネイル `Image` / 失敗時フォールバック（灰色枠＋ラベルのみ） |
| `ui/preset_picker/mod.rs` または `screen.rs`（変更） | 起動時または初回表示でサムネイルリソースを確保する **フック**（`Startup` ではなく egui パス内で遅延ロードする等、WASM との兼ね合いで実装時決定） |

**置かないもの**

- **PNG バイナリの生成スクリプト** — 必要なら `tools/` やメモのみ。Phase 3 の受け入れは「同梱アセットが読めること」が主。
- **`fractal_presets` の数式定義** — サムネは見た目用アセットであり、各 `build()` モジュールには手を入れない（マッピング表は `thumbnails.rs` か定数テーブル 1 ヶ所）。

## 受け入れ条件

- [ ] 各 `FractalPreset` に対応するサムネイルが格子に表示される（「新規」は仕様どおり）。
- [ ] WASM・ネイティブの両方で画像が読み込めない環境になった場合のフォールバック（ラベルのみ表示等）がある、または読み込み失敗が起きないことが検証されている。
- [ ] バイナリ／初回読み込みサイズがプロジェクトとして許容範囲である（ README または内部メモで目安が分かればよい）。
- [ ] `assets/` のサムネイル配置と `ui/preset_picker/thumbnails.rs` の責務が上表どおり分離されている。
